The problem

Agent instructions, skills, and permissions are scattered across tool-specific configs. Nothing is portable. Nothing composes.

The standard

AES gives your agentic projects a single, structured directory — portable across tools, shareable via registry, composable by design.

terminal 
# Install the CLI
$ pip install aes-cli

# Scaffold a new AES project
$ aes init

# Validate your .agent/ directory
$ aes validate

# Search the registry
$ aes search "deploy"

# Install a skill from the registry
$ aes install aes-hub/deploy@^1.0.0
How it works

Everything lives in .agent/

A single directory. Every agent artifact — instructions, skills, permissions, memory — organized and portable.

.agent/
  agent.yaml          # Project manifest
  instructions.md     # Agent instructions
  permissions.yaml    # Tool access rules
  skills/             # Composable capabilities
    deploy.skill.yaml
    deploy.md
  workflows/          # State machines
  memory/             # Persistent context
  commands/           # Runbook triggers
  registry/           # Vendored packages
