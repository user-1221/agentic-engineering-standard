The problem

Every session starts from zero

Your agent doesn't know your architecture, conventions, or deployment. You re-explain every session.

The standard

Every session builds on the last

AES gives your agent structured memory, real skills, and deep project context. Checked into git, portable across tools.

What you're building

Your project's .agent/ brain

Identity, skills, memory, and permissions — one directory, checked into git.

.agent/
  agent.yaml          # Who the agent is
  instructions.md     # How it thinks about your project
  permissions.yaml    # What it's allowed to do
  skills/             # What it knows how to do
    deploy.skill.yaml
    deploy.md
  workflows/          # How work flows through stages
  memory/             # What it's learned over time
  commands/           # Complex tasks it can run
  registry/           # Installed community skills
terminal 
# Install in seconds
$ pipx install aes-cli

# Give your project structured agent context
$ aes init

# Make sure your agent setup is complete
$ aes validate

# Find proven skills others have built
$ aes search "deploy"

# Add deployment expertise to your agent
$ aes install aes-hub/deploy@^1.0.0
How it works

Three steps to a smarter agent

1

Initialize

aes init scaffolds a .agent/ directory tailored to your project

2

Develop

Your agent learns your codebase. Memory builds. Skills get refined every session

3

Share

Publish skills to the registry. Install templates. Reuse across every project

Why it matters

Built for how you actually work

For individual developers

Your agent becomes a specialist

  • Templates give your agent domain expertise out of the box — ML, web, DevOps, research
  • Install proven skills from the registry. One command, zero learning curve
  • Memory persists across sessions. Every conversation makes your agent smarter
  • Works across Claude, Cursor, Copilot, Windsurf — switch without starting over

For teams

Every AI works from the same architecture

  • Every agent works from the same architecture, conventions, and design principles
  • New teammate? aes init --from your-team/template — instant project context
  • Instructions, permissions, and skills live in .agent/, always in sync via git
  • Ship independently. Everything fits because agents share the same foundation
