# Command: /feature

End-to-end feature development from requirements to deployment.

## Phase 1: Understand Requirements

Ask the user: What does the feature do? What data does it need? Any billing implications?

## Phase 2: Scaffold

Generate migration, API route, component, and test stubs.

## Phase 3: Implement

Write the code. Server components first, client only when interactive.

## Phase 4: Test (DO NOT SKIP)

Run unit → integration → e2e. Fix any failures before continuing.

## Phase 5: Deploy

Staging first, verify, then production with feature flag.
